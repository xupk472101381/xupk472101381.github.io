IOS接入详细流程

Unity导出Xcode时需要注意：

当需要导出Xcode工程时

  1. 小版本号：当游戏热更量较大或者渠道要求无热更的时候需要改动。通常新的版本号问下凯哥（有后台账号去后台查看）方法如下：

  1. 热更开关：默认打开（无特殊情况）

  2. 包类型：IOS_P越狱，IOS_L正版包，一般情况只用到这两种。

  3. 平台：固定IOS平台

  4. 外网测试包：勾选即为打白包。正式包不勾选。

  5. 当GameDataSetting文件修改完时，执行Export
    All（一般是改动什么文件Export什么文件），正版包还需要执行Export
    GarbageCode    和Export GarbageAsset如图所示：


    执行完成后我们就导出Xcode工程到本地

越狱包：

越狱包可以用同一个Xcode工程，所以我们可以导出一个Xcode工程共用。出不同的渠道包拷一个去接入即可。

正版包：

正版包不可以共用Xcode工程，所以每次出正版包我们需要导出一次Xcode工程，切记正版包还需要执行Export
GarbageCode和Export GarbageAsset
然后才能导出Xcode工程，每一个包单独用一个Xcode工程。

2.IOS文件，注意目录层级（Classes和Resources）Classes是工程的SDK接入的主要代码，Resources文件可以放SDK库文件，也可以放工程的根目录。具体路径如下图IOS/Resources/cl43Lczz/lib，SDK文件也可以放在工程根目录，指定到Unity-iPhone

在unity-iPhone ==> Build Setting ==>Other Linker Flags
加入-ObjC，指定Info.plist文件（在之前说的Resources文件下，或者用根目录的）。同时根据SDK文档配置Info.plist文件（苹果的相机，相册，麦克风权限，和SDK第三方的支付需要在此配置，需要注意。不明白的可以参考之前结果的工程）

关闭Bitcode，unity-iPhone ==> Build Setting ==>Enable Bitcode改No

4）接下里修改各项目的登陆请求地址，每个项目的地址都不一样，具体问PHP要。IOS/Classes/Common/CommonAppDelegate.mm**

三、然后开始接入SDK逻辑，我们需要参考SDK接入文档和SDK
Demo同步接入。具体逻辑写在IOS/Classes/NoSDK/AppDelegate.mm文件中，NoSDK是可以改名的（可以改成渠道或者SDK名字）。

具体需要接入以下几个接口：

  1. return 0; 其中0是渠道标识，运营提供

  2. Init(); 固定这么写

  3. Login();
    游戏的登录逻辑，在这里调用SDK的登录逻辑，将SDK返回的登录成功的数据和PHP通信。（一般写在登录成功回调里，SDK没有登录成功的回调则写在登录方法Login();里），其中    @”package”
    @”user_id” @”token”
    @”protocol”是发给PHP去登录验证的，参数名由PHP定义，不明白的地方可以参考之前的工程。

  4. LoginOut();
    游戏的登出/注销/切换账号逻辑，在这里调用SDK的登出/注销/切换账号逻辑。如果该SDK没有登出逻辑，则调用    U3DSendMessage(ON_LOGOUT,
    @””); 通知游戏登出即可。某些SDK有登出回调，就把U3DSendMessage(ON_LOGOUT,
    @””); 写在SDK的登录回调中。

  5. ShowCenter(); 预留接口。不用写逻辑。

  6. Pay();
    支付接口，SDK的支付方法写在这里，然后传相应的参数就行。有些项目有订单类型参数，就在后面定义多一个orderType

    支付参数释义 ：

    money购买商品价格 productId产品ID oderType订单类型

  1. Application(): SDK的初始化可以写在这里

  2. UpdatePlayInfo();
    更新角色信息接口，创建角色接口。SDK的更新角色接口和创建角色接口写在这里，下图为创建角色时调用。

  3. UpdateLevel(); 更新角色等级时调用该接口。

    游戏内的一些参数如图，这些参数会在登录，支付，上报信息等接口使用，具体调用方法示例：
    [instance mRoleID];

    从上往下依次释义

    mRoleID角色ID mNikeName角色名称 mRoleLvl角色等级

    mServerID服务器ID mServerName服务器名称 mGold货币余额

    mVipLvl vip等级 mFactionName公会名称 mCreateRoleTime创角时间

    mState当前状态 mYapType 支付方式 mDomain 域名（基本不用）

5）逻辑接入完成开始调试，联合PHP调通登录和支付即可。跑一下游戏没问题之后准备打包，直接执行Product==>Archive
